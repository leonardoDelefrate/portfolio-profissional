## Bitrem Workflow Implementation (2023–2024)

**Description:**
Implementation of a new workflow for transportation using bitrem vehicles, which was not previously supported by the legacy system. The functionality covers the entire process from freight creation to delivery completion, including fiscal document issuance (MDFe), freight calculation, vehicle closing, and tax validations.

**Technologies:**

* Backend: .NET Core, C#, Entity Framework, SQL Server
* Frontend: Windows Forms
* Databases: Oracle PL/SQL, PostgreSQL
* REST APIs
* Legacy system integration
* Excel and PDF export
* Advanced filters and dynamic tables

**Impact:**

* Significantly increased the profitability of bitrem operations
* Drastically reduced tax-related issues caused by the previous absence of this workflow in the system

## Insured Cargo Value Limit (2024)

**Description:**
Development of a feature to limit the total value of cargo transported per vehicle based on insurance policy coverage. The system blocks vehicle release when the value exceeds the allowed limit and automatically notifies the responsible department, without alerting operators directly to avoid security risks.

**Technologies:**

* Backend: .NET Core, C#, Entity Framework, SQL Server
* Frontend: Windows Forms, Angular
* Database: Oracle PL/SQL
* REST APIs
* Legacy system integration
* Automated email notifications

**Impact:**

* Strengthened security control for high-value cargo transportation
* Reduced the risk of losses and theft due to excess transported value

## IBS/CBS Tax Reform Implementation (2025–2026)

**Descrição:**
Development and adaptation of systems to comply with the Brazilian tax reform that introduced IBS (Imposto sobre Bens e Serviços) and CBS (Contribuição sobre Bens e Serviços), replacing several existing taxes nationwide. The project required updating business rules, tax calculations, validations, and integrations to ensure full legal compliance across all Rodonaves companies operating in Brazil.

This implementation impacted critical processes such as freight pricing, invoicing, fiscal document generation, tax breakdowns, and reporting, demanding high reliability, performance, and consistency across legacy and modern systems.

**Technologies:**

* Backend: .NET Core, C#, Entity Framework, SQL Server
* Frontend: Windows Forms, Angular
* Databases: Oracle PL/SQL, PostgreSQL
* REST APIs
* Legacy system integration
* Advanced search system
* Advanced filtering

**Impact:**

* Ensured full compliance with the new national tax model (IBS/CBS) across all company branches in Brazil
* Reduced fiscal risks and potential penalties through correct and standardized tax calculation
* Increased system reliability for high-volume fiscal operations during the tax transition period

## Criação do Fluxo Bitrem (2023–2024)

**Descrição:**
Implementação de um novo fluxo para o transporte com veículos do tipo bitrem, que ainda não era suportado pelo sistema legado. A funcionalidade cobre desde a criação do frete até a finalização da entrega, incluindo emissão de documentos fiscais (MDFe), cálculo de frete, fechamento de veículo e validações fiscais.

**Tecnologias:**

* Backend: .NET Core, C#, Entity Framework, SQL Server
* Frontend: Windows Forms
* Banco de Dados: Oracle PL/SQL, PostgreSQL
* APIs REST
* Integração com sistemas legados
* Exportação em Excel e PDF
* Filtros avançados e tabelas dinâmicas

**Impacto:**

* Aumentou significativamente a rentabilidade da operação de bitrems
* Reduziu drasticamente problemas fiscais por conta da ausência anterior desse fluxo no sistema

## Criação do Limite de Carga Segurada (2024)

**Descrição:**
Desenvolvimento de uma funcionalidade para limitar o valor total das cargas transportadas por veículo, com base na apólice de seguro. O sistema bloqueia a liberação de veículos quando o valor ultrapassa o limite e notifica automaticamente a área responsável, sem alertar os operadores diretamente para evitar riscos de segurança.

**Tecnologias:**

* Backend: .NET Core, C#, Entity Framework, SQL Server
* Frontend: Windows Forms, Angular
* Banco de Dados: Oracle PL/SQL
* APIs REST
* Integração com sistemas legados
* Envio automático de e-mails

**Impacto:**

* Reforçou o controle de segurança no transporte de cargas de alto valor
* Reduziu o risco de perdas e furtos por excesso de valor transportado
